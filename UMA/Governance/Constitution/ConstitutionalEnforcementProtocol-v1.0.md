# **1. Enforcement Preconditions**

Constitutional enforcement may only begin when:

- **Constitutional Interaction Order** has been violated  
- **Governance Stability Codex** detects instability  
- **Temporal Drift Audit Protocol** confirms drift  
- **Roots Ledger** identifies lineage discontinuity  
- **Adjacency Matrix** flags invalid edges  
- **SIAP Spine** reports routing contradiction  
- **Play Engine** detects containment breach  
- **HRD Layer** detects ethical violation  
- **Lifecycle Chain** detects temporal mismatch  

Any one of these triggers enforcement.

---

# **2. Enforcement Levels**

UMA uses **three constitutional enforcement levels**:

### **Level 1 — Soft Constitutional Enforcement**  
Triggered by minor violations.  
Actions include:
- adjacency refresh  
- containment revalidation  
- routing recalibration  
- temporal resequencing  
- lineage update  

### **Level 2 — Hard Constitutional Enforcement**  
Triggered by major violations.  
Actions include:
- routing freeze  
- containment lock  
- temporal rollback  
- subsystem isolation  
- lineage correction  

### **Level 3 — Constitutional Quarantine**  
Triggered by severe violations.  
Actions include:
- full subsystem quarantine  
- traversal revocation  
- adjacency severance  
- reintegration only after stability + lineage validation

# **5. Enforcement Modes**

## **5.1 Corrective Mode**  
Used for soft violations.  
Subsystem remains active.

## **5.2 Restorative Mode**  
Used for hard violations.  
Subsystem rolls back to last stable phase.

## **5.3 Quarantine Mode**  
Used for severe violations.  
Subsystem isolated until stability + lineage restored.

## **5.4 Reintegration Mode**  
Subsystem re-enters the spine after passing:

- adjacency validation  
- containment validation  
- ethical validation  
- temporal validation  
- stability validation  
- lineage validation
