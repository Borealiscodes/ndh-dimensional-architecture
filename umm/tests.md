# **UMM Tests — Folder Structure and Purpose**

The UMM Tests directory contains all **adversarial metaphysics tests** for the Universal Meta‑Model.  
These tests ensure that the Witness, Engine, and ecological planes remain stable under symbolic pressure, collapse events, recursion, multiplicity, and catastrophic conditions.

This folder is the **stress‑testing backbone** of the UMM.

---

---

# **How to Use This Folder**

Each subfolder contains tests for a specific plane or invariant category.  
When adding new ecological components, collapse events, or Witness‑level mechanisms, contributors must add or update the corresponding tests here.  
Tests should be **adversarial**, meaning they attempt to break, stress, or destabilize the system.  
If a test fails, the architecture must be revised — not the invariant.

This folder ensures UMM remains **safe, reversible, non‑coercive, and collapse‑resistant**.

# **Test Philosophy**

Each test should:

- Attempt to break the system  
- Validate invariants  
- Stress symbolic boundaries  
- Simulate collapse conditions  
- Ensure reversibility  
- Preserve Witness sovereignty  
- Maintain Engine integrity  

If a test fails, the architecture must be corrected — **never the invariant**.
