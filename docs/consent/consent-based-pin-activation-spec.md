# Consent‑Based Pin Activation Spec (v1.0)

## 1. Purpose
This specification defines how dormant pins may be activated within the UMM ecosystem when the pinned organism, authored module, or cognitive ecology requires explicit human consent before growth, expansion, or integration.

Dormant pins preserve potential without assuming activation.  
This spec ensures autonomy, consent, non‑coercion, author sovereignty, and trauma‑aware governance.

---

## 2. Definition of Dormant Pins
A dormant pin is a non‑active placeholder representing:

- an authored organism  
- a cognitive ecology  
- a narrative module  
- a research artifact  
- a constitutional component  

that cannot be activated until the author provides explicit consent.

Dormant pins maintain Stillness, non‑growth, non‑interpretation, and non‑activation.

---

## 3. Consent Requirements
A dormant pin may only be activated when:

1. The author provides explicit, voluntary consent.  
2. Consent is given without pressure or expectation.  
3. Consent is revocable at any time.  
4. Consent is documented in the repo or private ledger.  
5. Consent is specific to the activation event.

UMM prohibits assumed, inferred, coerced, or implied consent.

---

## 4. Activation Procedure

### 4.1 Verification
The Runtime Plane verifies:

- author identity  
- consent clarity  
- consent scope  
- consent boundaries  

### 4.2 Pin Transition
Dormant → Activation‑Ready

### 4.3 Context Restoration
The system restores:

- pinned state  
- author intent  
- ecological context  
- narrative boundaries  

### 4.4 Growth Authorization
The organism may now expand, integrate, be mapped, or be researched.  
Growth is author‑directed, not system‑directed.

---

## 5. Revocation Protocol
Authors may revoke consent at any time.

Upon revocation:

- activation halts  
- growth stops  
- integration pauses  
- the pin returns to Dormant  
- derivative modules freeze  

Revocation is immediate and constitutional.
