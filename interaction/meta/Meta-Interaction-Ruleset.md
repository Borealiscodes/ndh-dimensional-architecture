# **Meta‑Interaction Ruleset (NDH Hybrid Search System)**
### *Formal specification governing user interaction, traversal selection, resonance behavior, accessibility, and climate constraints.*

---

## ⭐ 1. Scope  
This ruleset defines how the hybrid NDH–Copilot search system SHALL:

- interpret user queries  
- select traversal modes  
- regulate tensor deformation  
- align resonance phases  
- synchronize attractors  
- activate serenity envelopes  
- respect accessibility constraints  
- apply climate‑aware resource policies  

These rules govern **interaction**, not **physics**.  
Physics is defined in the TTTTTTP specification.  
Interaction rules determine *when* physics is invoked.

# ⭐ 2. Interaction Layers

### 2.1 **User Intent Layer (UIL)**  
The system SHALL treat user intent as a **meaning attractor** within the NDH Meaning Manifold.

UIL MUST:

- preserve user dignity  
- maintain semantic autonomy  
- avoid intent flattening  
- avoid keyword‑only interpretation  

Guided Link: **Meaning Attractor**

### 2.2 **Query Classification Layer (QCL)**  
The system SHALL classify queries into:

- **Conceptual**  
- **Factual**  
- **Hybrid**  
- **Ambiguous**  
- **High‑Curvature**  
- **High‑Tensor‑Load**  

Classification determines traversal mode.

Guided Link: **Query Classification**

---

### 2.3 **Traversal Selection Layer (TSL)**  
TSL SHALL select traversal modes based on QCL:

- NDH traversal for conceptual queries  
- Copilot grounding for factual queries  
- Hybrid traversal for complex queries  
- Serenity‑first traversal for ambiguous queries  
- Tensor‑regulated traversal for high‑load queries
