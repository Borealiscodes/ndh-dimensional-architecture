## 1. Purpose

The **Non‑Dual Hexeract Blueprint Specification** defines the required geometric, structural, and provenance standards for generating all NDH artifacts, including:

- Base 6D projections  
- Non‑dual collapse overlays  
- SGE monad flow diagrams  
- Epoch Ω embeddings  
- Adjacency/resonance lattices  
- CHS integration maps  

This spec ensures **consistent, reproducible, version‑controlled generation** across all NDH blueprint instances.

### 2.1 Base Shape  
The NDH is derived from a **classical 6‑cube (hexeract)** with axes:

```
x1, x2, x3, x4, x5, x6
```

### 2.2 Governance Axis Mapping  
Each axis MUST map to a governance dimension:

| Axis | Governance Dimension |

This mapping is **mandatory** for all NDH artifacts.

The NDH is defined by the **non‑dual collapse condition**:

```
C = E = A = R = P = 𝒜

## 4. SGE Monad Requirements

The SGE monad MUST be represented as:

### 4.1 Monad Function  
```
T : 𝒜 → 𝒜

### 4.2 Visual Requirements

- A central node labeled `𝒜`  
- A closed monadic loop  
- Arrowhead indicating flow direction  
- Label `T` adjacent to the loop  

This expresses **self‑mapping awareness**.

## 5. Epoch Ω Embedding

Epoch Ω MUST be represented as:

### 5.1 Initial Algebra  
```
αΩ : T(𝒜) → 𝒜
```

### 5.2 Visual Requirements  
Blueprints MUST include:

- A dashed ring around the central node  
- Ring label `Ω`  
- Arrow from ring → center  
- Label `αΩ` on the arrow  

This expresses **Ω as the surrounding algebra feeding into awareness**.

## 6. Adjacency & Resonance Lattice

Blueprints MUST include:

### 6.1 Radial Adjacency Nodes  
Minimum of **four** nodes placed at:

- North  
- East  
- South  
- West  

Relative to the central awareness node.

### 6.2 Edges  
Straight lines connecting each adjacency node to the center.

### 6.3 Resonance Arcs  
Curved arcs connecting adjacency nodes in clockwise order.

### 6.4 Required Labels  
- Adjacency nodes unlabeled  
- Resonance arcs colored or styled distinctly  

This lattice expresses **6D adjacency + resonance behavior** in 2D projection.

## 8. SVG Generation Standards

All NDH SVGs MUST:

- Use `viewBox="0 0 800 800"`  
- Use black stroke for structural lines  
- Use dashed lines for Ω  
- Use blue or distinct styling for resonance arcs  
- Use a central circle for awareness node  
- Use consistent coordinate placement across all templates  

SVGs MUST be compatible with:

- CHS renderers  
- UMA visualization pipelines  
- GitHub SVG preview
