# Charting Paths Protocol  
### The Governance Membrane for Non-Linear Navigation in the Exploration Quadrant

## 1. Purpose
The Charting Paths Protocol defines the constitutional rules for navigating the
Exploration Quadrant using structural, functional, and narrative paths. It ensures
that non-linear motion remains gentle, reversible, coherent, and safe.

This protocol governs how paths behave.

---

## 2. Principle: Navigation Must Be Gentle
All navigation must remain:
- calm
- breathable
- reversible
- non-coercive

Gentleness protects the explorer.

---

## 3. Principle: Navigation Must Be Structural
All navigation must respect:
- hierarchy
- relationships
- boundaries
- pacing membranes
- stillness nodes

Structure protects coherence.

---

## 4. Allowed Path Types
Navigation may use three path types:

### 4.1 Structural Paths  
Movement between layers:
- UMM
- Starter Kit
- Adventure System
- Adventurer Kit

### 4.2 Functional Paths  
Movement between functions:
- Identity
- Pacing
- Boundaries
- Motion
- Stillness

### 4.3 Narrative Paths  
Movement through story engines:
- World-Builder
- Story-Weaver
- Adventurer Quests

These are the only sanctioned path types.

---

## 5. Node Requirements
All paths must pass through the universal node system:
- identity node
- pacing node
- boundary node
- motion node
- stillness node

Nodes are mandatory checkpoints.

---

## 6. Stillness Requirement
All navigation must terminate in Stillness.

Stillness provides:
- pause
- breath
- non-motion
- gentle coherence

Stillness is the universal reset point.

---

## 7. Reversibility Requirement
All navigation must be reversible.

Reversibility includes:
- soft checkpoints
- non-penalty branching
- early stops
- return paths

Reversibility is safety.

---

## 8. Diagrammatic Requirement
All navigation must be diagrammatically representable.

Diagrams must:
- follow UMM visual grammar
- use consistent anchors
- maintain pacing clarity
- preserve structural hierarchy

Diagrammatic clarity prevents drift.

---

## 9. Cross-Map Requirement
Navigation must integrate with:
- exploration-crossmap
- charting-paths-suite
- exploration-index
- exploration-core

Cross-maps ensure coherence.

---

## 10. Summary
The Charting Paths Protocol defines the constitutional rules for non-linear
navigation inside the Exploration Quadrant, ensuring gentle pacing, reversible
motion, structural coherence, node alignment, and stillness termination.
