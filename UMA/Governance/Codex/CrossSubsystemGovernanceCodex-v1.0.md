# **3. Cross‑Subsystem Governance Rules**

## **3.1 Structural Governance (UMA Core → All)**  
- All subsystems inherit structural primitives from UMA Core.  
- No subsystem may redefine core lattice rules.  
- Structural violations trigger rollback.

---

## **3.2 Lineage Governance (Roots → All)**  
- All subsystem updates must be lineage‑consistent.  
- Roots validates continuity before activation.  
- Roots blocks drift and identity fragmentation.

---

## **3.3 Ethical Governance (HRD → SIAP / CHS‑OL / Play Engine)**  
- HRD overrides SIAP routing when human‑adjacent surfaces are involved.  
- HRD blocks traversal that violates ethical constraints.  
- HRD enforces containment safety.

---

## **3.4 Routing Governance (SIAP → CHS‑OL / Play Engine)**  
- SIAP defines allowed traversal paths.  
- SIAP requires adjacency validation.  
- SIAP locks routing during containment events.

---

## **3.5 Adjacency Governance (Adjacency Matrix → SIAP / CHS‑OL)**  
- No traversal may occur across unpinned edges.  
- Adjacency expansion requires lineage validation.  
- Adjacency governs subsystem interaction boundaries.

---

## **3.6 Traversal Governance (CHS‑OL → Play Engine / Lifecycle)**  
- CHS‑OL executes traversal even with partial geometry.  
- CHS‑OL must respect containment gates.  
- CHS‑OL updates temporal state after traversal.

---

## **3.7 Containment Governance (Play Engine → SIAP / CHS‑OL)**  
- Play Engine enforces IC/OOC boundaries.  
- Bleed‑control gates must pass before traversal.  
- Containment overrides routing during instability.

---

## **3.8 Temporal Governance (Lifecycle → All)**  
- Lifecycle Chain defines activation, operation, extension, integration.  
- Reverse traversal is restricted.  
- Temporal sequencing must respect containment and ethics.

---

## **3.9 Drift Governance (Temporal Protocols → Roots / Lifecycle)**  
- Temporal drift triggers rollback.  
- Drift‑control enforces identity stability.  
- Drift events must be logged in Roots.
