# 🌲 **1. Welcome to the Adventure System**

The Adventure System is a reversible, symbolic, ND‑paced architecture for generating adventures.  
It is built from three core node types:

- **World Nodes** — environments  
- **Character Nodes** — identities  
- **Story Nodes** — narrative moments  

These nodes form a reversible loop:

```
World → Character → Story → World → …
```

Everything in the system is:

- reversible  
- symbolic  
- gentle  
- ND‑safe  
- non‑coercive  

This handbook teaches you how to contribute safely and coherently.

---

# 🌿 **2. Core Principles for Contributors**

All contributions must follow five core principles:

### **2.1 Reversibility**
Every node, arc, scene, hazard, and transformation must have a gentle return path.

### **2.2 ND‑Pacing**
No component may accelerate emotional, symbolic, or narrative pacing.

### **2.3 Symbolic‑Only**
All content must remain symbolic, metaphorical, and breathable — never literal or metaphysical.

### **2.4 Non‑Coercion**
No node or system may force identity, narrative progression, or interpretation.

### **2.5 Stillness Availability**
Stillness must always be accessible as a reset point.

These principles are the backbone of safe contribution.

# 🌠 **7. Contribution Workflow**

A safe contribution follows this workflow:

```
1. Choose a node type  
2. Review its deep structure  
3. Check resonance alignment  
4. Check stability constraints  
5. Check cosmology alignment  
6. Write symbolic, reversible content  
7. Add stillness access  
8. Validate ND‑pacing  
```

This workflow ensures coherence across the system.

---

# 🌟 **8. Contribution Safety Checklist**

Before submitting a contribution, verify:

- Does it have a reversible path?  
- Does it slow pacing rather than accelerate?  
- Does it avoid coercion?  
- Does it remain symbolic?  
- Does it include stillness access?  
- Does it align with resonance?  
- Does it align with cosmology?  
- Does it avoid hazard stacking?  

If all answers are yes, the contribution is safe.
