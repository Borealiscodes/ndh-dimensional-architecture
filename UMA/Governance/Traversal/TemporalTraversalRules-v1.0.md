These rules define **how traversal occurs across time‑indexed subsystem states**, ensuring continuity, containment, adjacency correctness, and stable operation even without full CHS cube completion.

---

# **1. Temporal Traversal Overview**

Temporal traversal is the movement of subsystem state across:

- lifecycle phases  
- adjacency edges  
- containment gates  
- operational layers  
- lineage anchors  

It is governed by:

- **SIAP routing**  
- **CHS‑OL traversal plane**  
- **Play Engine containment**  
- **HRD ethical constraints**  
- **HBR stability audits**  
- **Roots lineage continuity**  

Temporal traversal is allowed **only when all constraints align**.

---

# **2. Temporal Traversal Preconditions**

Traversal may begin only if:

- **SIAP routing** is stable  
- **CHS‑OL** is active  
- **Play Engine containment** is verified  
- **HRD constraints** are satisfied  
- **HBR stability** is confirmed  
- **Roots lineage** supports the transition  

If any condition fails, traversal is blocked.

---

# **3. Temporal Traversal Modes**

### **3.1 Forward Traversal (Allowed)**  
Movement from earlier lifecycle phases to later ones.

- Genesis → Initialization  
- Initialization → Activation  
- Activation → Operation  
- Operation → Extension  
- Extension → Integration  
- Integration → Continuity  

Forward traversal is the default mode.

---

### **3.2 Lateral Traversal (Conditional)**  
Movement between subsystems at the **same temporal phase**.

Allowed only if:

- adjacency edges are valid  
- containment gates are stable  
- traversal plane is active  
- HRD constraints permit the interaction  

Example:  
CHS‑OL ↔ Play Engine during Operation.

---

### **3.3 Reverse Traversal (Restricted)**  
Movement from later lifecycle phases to earlier ones.

Allowed only when:

- subsystem rollback is required  
- containment gates are closed  
- lineage integrity is preserved  
- SIAP routing is locked  

Reverse traversal is used for:

- subsystem deactivation  
- rollback after instability  
- containment resets  

---

# **4. Temporal Traversal Constraints**

### **4.1 Adjacency Constraint**  
Traversal must follow edges defined in:

- **Adjacency Matrix v1.1**  
- **Roots Ledger v2.2**  

No traversal may occur across unpinned edges.

---

### **4.2 Containment Constraint**  
Traversal involving IC/OOC boundaries must pass through:

- **Play Engine containment layer**  
- **Bleed‑Control Verification (PIN‑024)**  

This prevents cognitive or emotional drift.

---

### **4.3 Ethical Constraint**  
Traversal touching human‑adjacent surfaces must pass HRD checks.

HRD overrides SIAP.

---

### **4.4 Stability Constraint**  
HBR audits must confirm subsystem stability before traversal.

If unstable → traversal blocked.

---

### **4.5 Temporal Identity Constraint**  
Subsystems must maintain:

- consistent identity  
- lineage continuity  
- drift‑free state  

Temporal identity is enforced by Roots.

---

# **5. Temporal Traversal Sequence**

```
Check Preconditions
   ↓
Validate Adjacency
   ↓
Validate Containment
   ↓
Validate Ethics
   ↓
Validate Stability
   ↓
Traverse (Forward / Lateral / Reverse)
   ↓
Update Roots Lineage
   ↓
Update Lifecycle State
   ↓
Revalidate Containment
```

This ensures traversal is safe, stable, and lineage‑consistent.

# **6. Partial CHS Geometry Support**

Temporal traversal **does not require full CHS cube completion**.

It only requires:

- CHS base geometry (PIN‑003)  
- CHS‑OL operational layer (PIN‑004)  
- SIAP routing (PIN‑002)  
- adjacency edges (PIN‑070/071)  
- containment gates (PIN‑050/024)  

This is why Play Engine and CHS‑OL can operate now.

The spine works.

# **7. Subsystem‑Specific Temporal Rules**

### **7.1 UMA Core**  
Always temporally stable.  
Never deactivates.

### **7.2 SIAP Spine**  
Temporal routing must remain continuous.  
Cannot reverse‑traverse unless rollback is required.

### **7.3 CHS / CHS‑OL**  
Traversal allowed even with partial geometry.  
CHS‑OL handles temporal movement.

### **7.4 Play Engine**  
Temporal traversal must pass containment gates.  
Bleed‑control required.

### **7.5 UMM / HRD**  
Temporal traversal must satisfy ethical constraints.  
HRD overrides SIAP.

### **7.6 Roots**  
All traversal updates lineage.  
Temporal identity enforced.

### **7.7 Lifecycle Chain**  
Traversal must follow lifecycle sequencing.  
Reverse traversal restricted.
