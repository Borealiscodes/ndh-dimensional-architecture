## ⭐ **How to Use This Protocol**  
This section tells developers, players, designers, and facilitators exactly how to operate the entire system safely and effectively.

### **1. Start With IC/OOC Clarity**  
Before any scene, quest, or mode:

- Activate **IC_ON()** when embodying a character.  
- Activate **IC_OFF()** when speaking as yourself.  
- Use **META_PAUSE()** whenever you need a moment.  
- Use **DECOMPRESS()** after intense scenes.

This keeps imagination expansive but boundaries intact.

- **Choose Your Own Adventure**

- **Real‑Time Strategy**  
- **Dungeon Siege Troupe Mode**  

Each mode has its own safety defaults and decompression recommendations.

### **4. Apply Ethical Design Principles**  
Before adding anything to the repo:

- Check consent requirements  
- Tag heavy themes  
- Add decompression routes  
- Ensure IC/OOC boundaries are clear  
- Frame consequences, never ambush them  

This keeps imagination powerful but never harmful.

---

### **5. Use Player‑Facing Safety Tools**  
Players can adjust:

- theme filters  
- intensity sliders  
- decompression prompts  
- IC/OOC toggles  
- Meta Pause button  
- “End Scene Safely” flow  

This gives players agency over their own cognitive safety.

For groups:

- Use the consent matrix

- Run troupe‑wide decompression after heavy scenes  
- Reaffirm OOC relationships  

This keeps collaborative imagination safe and fun.

### **7. End Every Session With Closure**  
Use:

- **DECOMPRESS()**  
- IC/OOC separation  
- emotional sorting  
- grounding  
- “Scene complete” or “Session complete”  

This prevents bleed and protects long‑term wellbeing.
