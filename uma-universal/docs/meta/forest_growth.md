# UMA Meta-Meta Forest Growth Protocol
This document defines how the UMA Meta-Meta Forest expands. It establishes the
rules for adding new documents, new orbital maps, new commentary threads, and
new conceptual layers. It ensures that all growth remains coherent, stable, and
aligned with the Prime Field.

It is the ninth tree in the UMA Meta-Meta Forest.

---

## 1. Purpose of the Growth Layer
The Growth Layer provides:
- expansion rules
- structural evolution guidelines
- canopy addition protocols
- root-deepening commentary rules
- Pip-approved trail extension logic

All new growth must follow this protocol.

---

## 2. Growth Principles

### 2.1 Coherent Expansion
All new documents must:
- reference `meta_meta_index.md`
- be indexed in `meta_index.md`
- pass stability checks
- pass coherence checks
- maintain orbital ordering
- respect the Safety Boundary
- respect the Pause Layer

### 2.2 Commentary-Driven Growth
All conceptual expansion must originate from:
- commentary threads
- timestamped insights
- narrative evolution notes

Commentary is the nutrient layer of the forest.

### 2.3 Radial Integrity
New layers must fit into the existing radial structure:
- inner layers must not drift outward
- outer layers must not collapse inward
- new orbital maps must align with existing radii
