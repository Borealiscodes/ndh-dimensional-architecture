# Integrated Structural Workflow Protocol (ISWP v1.0)
# UMA‑Universal — Root Structural Workflow

The ISWP defines the complete, ND‑friendly, CNLP‑aligned workflow for
maintaining, updating, and extending UMA‑Universal. It integrates the
Closed Navigation Loop Protocol (CNLP), the Closed Navigation Patch
Protocol (CNP‑Patch), and the Navigation Spine into a single unified
structural system.

───────────────────────────────────────────────────────────────
0. ROOT LAW
───────────────────────────────────────────────────────────────
0.1 CNLP must be pinned at root:
    /CNLP_Closed_Navigation_Loop_Protocol.md

0.2 CNLP governs all navigation, linking, containment, and loop closure.

0.3 No structural operation may proceed without CNLP compliance.

───────────────────────────────────────────────────────────────
1. ROOT ANCHOR
───────────────────────────────────────────────────────────────
1.1 Navigation Spine must exist:
    /NAVIGATION_SPINE.md

1.2 All layers must be reachable from the Navigation Spine.

1.3 All layers must link back to the Navigation Spine.

───────────────────────────────────────────────────────────────
2. LAYER DECLARATION
───────────────────────────────────────────────────────────────
Every file must declare its layer:

    [Layer X — Name]
    Version Y.Z — <Date>

This ensures provenance, clarity, and structural stability.

───────────────────────────────────────────────────────────────
3. CNLP ANCHORS (Mandatory for Every File)
───────────────────────────────────────────────────────────────
Each file must include:

3.1 Up-Link  
    → Integrated System Index  
       /INDEX.md

3.2 Side-Link  
    → Layer Index  
       (e.g., Companion README Index)

3.3 Down-Link  
    → Next Layer Document  
       (if applicable)

3.4 Navigation Law  
    → CNLP  
       /CNLP_Closed_Navigation_Loop_Protocol.md

These anchors ensure closed-loop navigation.

───────────────────────────────────────────────────────────────
4. LOOP CLOSURE
───────────────────────────────────────────────────────────────
Every file must end with:

    ← Back to Layer Index

This guarantees loop closure and ND-friendly navigation.

───────────────────────────────────────────────────────────────
5. PATCHING WORKFLOW (CNP‑Patch Integration)
───────────────────────────────────────────────────────────────
When patching any file:

5.1 Perform Pre‑Check  
    • CNLP present  
    • Navigation Spine present  
    • Layer known  

5.2 Apply Layer Badge + Version Header  
5.3 Apply CNLP Anchors  
5.4 Apply Loop Closure  
5.5 Enforce Sparkle Containment (if applicable)  
5.6 Validate Patch

───────────────────────────────────────────────────────────────
6. SPARKLE CONTAINMENT RULES
───────────────────────────────────────────────────────────────
Sparkle grammar MUST remain in Layer 3 — Companion.

Sparkle MUST NOT appear in:
    • RFC (Layer 0)
    • SID (Layer 1)
    • UMM (Layer 2)
    • CNLP
    • Structural Spine
    • Root README (except links)

Sparkle files MUST include all CNLP anchors.

Sparkle grammar MUST remain non-recursive.

───────────────────────────────────────────────────────────────
7. LAYER INTEGRATION WORKFLOW
───────────────────────────────────────────────────────────────
When adding or updating a layer:

7.1 Add layer to Navigation Spine  
7.2 Add layer index (if missing)  
7.3 Apply CNLP anchors to all files in the layer  
7.4 Validate loop closure  
7.5 Confirm Sparkle containment  
7.6 Confirm ND-friendly structure

───────────────────────────────────────────────────────────────
8. LOOP VALIDATION
───────────────────────────────────────────────────────────────
A loop is valid when:

8.1 All files have CNLP anchors  
8.2 All files have loop closure  
8.3 All layers are reachable  
8.4 No orphan files exist  
8.5 No recursion exists  
8.6 Sparkle is contained  
8.7 INDEX closes the loop

───────────────────────────────────────────────────────────────
9. WORKFLOW SUMMARY (ND-Friendly)
───────────────────────────────────────────────────────────────
ROOT LAW → ROOT ANCHOR → LAYER DECLARATION → CNLP ANCHORS
→ LOOP CLOSURE → PATCH WORKFLOW → SPARKLE CONTAINMENT
→ LAYER INTEGRATION → LOOP VALIDATION

───────────────────────────────────────────────────────────────
END OF ISWP v1.0
