## 🛡️ Dungeon Siege–Style Troupe Mode  
*(Party‑based progression, multi‑character control, bleed‑safe design)*

### **Overview**  
Troupe Mode enables one player (or multiple players) to control a **party of characters** simultaneously, mirroring Dungeon Siege’s continuous movement and real‑time tactical flow.  
This mode distributes emotional load, reduces bleed, and supports IC/OOC switching per character.

---

## 🎮 1. Troupe Engine Structure

### **Party Composition**
- 2–6 characters  
- Each character has:
  - IC/OOC toggle  
  - bleed‑aware sheet  
  - emotional flags  
  - decompression profile  
  - consent matrix entry  

### **Control Modes**
- Direct Control — embody one character IC  
- Command Control — issue tactical orders OOC  
- Auto‑Resolve — engine handles minor actions  

---

## 🔀 2. Troupe‑Aware IC/OOC Toggle Macros

- **IC_ON(characterID)**  
- **IC_OFF(characterID)**  
- **META_PAUSE()**  
- **DECOMPRESS(target)**  

These allow switching characters without switching identities.

---

## 🧠 3. Bleed‑Aware Character Sheets (Troupe Edition)

Each sheet includes:

- Player Identity Anchor  
- Character Identity Anchor  
- Emotional Flags (G/Y/R)  
- Bleed Notes  
- Troupe Interaction Notes  

---

## 🧸 4. Decompression Ritual Packs (Troupe Edition)

### **Solo Decompression**  
For one character after a heavy scene.

### **Troupe Decompression**  
For party‑wide emotional or tactical intensity.

Includes:

- group grounding  
- shared recap  
- IC/OOC separation  
- emotional sorting per character  

---

## 🧬 5. Multiplayer Safety Scaffolding (Troupe Edition)

### **Consent Matrix**  
Per‑player and per‑character.

### **Safety Signals**  
Green / Yellow / Red.

### **Conflict Protocol**  
- IC conflict only with OOC consent  
- per‑character consent for multi‑character conflict  
- troupe‑wide repair check

- Shared XP Pool  
- Distributed Stakes  
- Rotating Spotlight  
- Death/Injury Protocol (OOC consent required)
