# ⭐ UMA Mandala Specification Note  
**Canonical geometric rules for all radial UMA diagrams**

---

## 🌐 1. Mandala Purpose  
The UMA Mandala expresses the architecture in its correct geometry:

- **Movement (Thoroughfare)**  
- **Runtime (Cosmology Engine)**  
- **Behavior (Meta Behavior)**  

These are not hierarchical layers — they are **concentric rings** of a single system.

The mandala is the parent geometry for:

- Interaction Mandala  
- Cosmology‑Only Mandala  
- Behavior Mandala  
- UMA Orbital Model

All sub‑mandalas must inherit the rules below.

---

## 🌀 2. Ring Specification  
### **Ring Radii (canonical)**
- **Ring 1 (Movement):** inner radius  
- **Ring 2 (Runtime):** middle radius  
- **Ring 3 (Behavior):** outer radius  

Spacing between rings must be **equal** to preserve symmetry.

### **Ring Order (fixed)**
1. **Movement Ring** — Corridors, Braid, Weave  
2. **Runtime Ring** — State, Flow, Transition  
3. **Behavior Ring** — Emergence, Structure, Absurdity  

This order is never reversed or rearranged.

### **Ring Semantics**
- Movement = *grammar of traversal*  
- Runtime = *grammar of state and flow*  
- Behavior = *grammar of emergence and interpretation*  

---

## 🔱 3. Spoke Specification  
### **Spoke Angles (canonical)**
- **0°** → Corridors → State → Emergence  
- **120°** → Braid → Flow → Structure  
- **240°** → Weave → Transition → Absurdity  

These angles are fixed and inherited by all sub‑mandalas.

### **Spoke Rules**
- Each spoke must contain **exactly three nodes**  
- Nodes must be placed at **equal radial intervals**  
- Spokes must be straight lines from center outward  
- Spokes must never curve, bend, or branch  

### **Spoke Semantics**
Each spoke expresses a **full UMA chain**:

Movement → Runtime → Behavior  
Corridors → State → Emergence  
Braid → Flow → Structure  
Weave → Transition → Absurdity  

---

## 🔘 4. Node Specification  
### **Node Count**
Each ring contains **three nodes**, always:

- Movement: Corridors, Braid, Weave  
- Runtime: State, Flow, Transition  
- Behavior: Emergence, Structure, Absurdity  

### **Node Placement**
- Nodes must be placed at **120° intervals**  
- Nodes must align radially with their spoke counterparts  
- Node size must remain consistent across rings  

### **Node Labels**
Labels must be centered and must not overlap rings or spokes.

---

## 🧭 5. Center Specification  
### **Center Rules**
The center may contain:

- “UMA Core”  
- or be left empty  

But it must remain:

- circular  
- unobstructed  
- visually stable  

The center is the **semantic anchor** of the mandala.

---

## 🧩 6. Inheritance Rules for Sub‑Mandalas  
Every sub‑mandala must inherit:

### **Geometry**
- ring radii  
- node angles  
- spoke angles  
- radial spacing  
- node alignment  

### **Semantics**
- Movement always inner  
- Runtime always middle  
- Behavior always outer  

### **Constraints**
Sub‑mandalas may **remove rings**, but may not:

- reorder rings  
- change angles  
- change spacing  
- change node count  
- change spoke alignment  

This ensures all diagrams remain interoperable.

---

## ⭐ 7. Canonical Mandala Family  
Once this specification is locked, the following diagrams can be generated cleanly:

- Interaction Mandala  
- Cosmology‑Only Mandala  
- Behavior Mandala  
- UMA Orbital Model

All will be consistent, readable, and structurally aligned.
